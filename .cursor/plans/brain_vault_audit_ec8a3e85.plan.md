# Brain Vault Audit — Full Plan

## Findings

### 1. Structural anomaly — rogue `index/` folder
A file landed in an undocumented `index/` directory at the vault root:

- `index/implementation/clawvis/2026-04-12-agent-banner-vs-wizard.md`

Every other implementation note lives in `inbox/daily/implementation/<slug>/`. This file needs to be **moved** there and the `index/` folder removed.

Side-effect: `inbox/connections/suggested-connections.md` was generated by an old audit run that used `index/implementation/...` paths instead of `inbox/daily/implementation/...`. Those references are stale and should be fixed (or the file regenerated on next audit).

### 2. Missing `caps/student.md`
Both `IDENTITY.md` and `index.md` reference `[[caps/student]]` but the file does not exist. The only cap files are `Artelys.md`, `developer.md`, `entrepreneur.md`. Either the file was never created or it was deleted. Needs to be created from the `projects/_template.md` pattern or the reference cleaned up.

### 3. `Local-Brain.md` — L1 table incomplete
The L1 row is missing `llm-wiki.md` (added today per `log.md`, but only the tree was updated, not the table):

**Current:**
```
| L1 | … | `IDENTITY.md`, `breadcrumbs.md`, `docs/memory/MEMORY.md`, `log.md` | ~7000 tokens |
```
**Should be:**
```
| L1 | … | `IDENTITY.md`, `llm-wiki.md`, `breadcrumbs.md`, `docs/memory/MEMORY.md`, `log.md` | ~7000 tokens |
```

### 4. `Local-Brain.md` — structure tree is stale
The tree shows folders that don't exist and omits folders that do:
- Shows `daily/consolidated/` → doesn't exist (notes live in `inbox/daily/`)
- Shows `resources/knowledge/patterns/` and `resources/knowledge/sops/` → neither exists
- Missing: `caps/`, `inbox/`, `raw/`, `meta/`, `breadcrumbs.md`, `llm-wiki.md`, `log.md`

The tree needs a full rewrite to match the actual directory listing.

### 5. `index.md` — broken wikilink
`[[caps/student]]` points to a non-existent file. Should either be removed or replaced with the correct cap if a student cap exists elsewhere.

### 6. `llm-wiki.md` — placement is correct
The file is at the vault root, tagged `L1`, and linked from both `index.md` and `Local-Brain.md`. **No move needed.** Minor improvement: the layer table in `llm-wiki.md` could mention `inbox/` and `meta/` under L2/L3 to complete the picture.

### 7. Cross-link gaps (orphan resources)
Many `resources/knowledge/` files have zero incoming wikilinks. High-value connections to add:
- `resources/knowledge/operational/claude-finops.md` → link from `breadcrumbs.md`
- `resources/knowledge/operational/claude-pitfalls.md` → link from `breadcrumbs.md`
- `resources/knowledge/architecture/openclaw-openrouter.md` → link from `projects/clawvis.md`
- `caps/entrepreneur.md` → add `[[caps/developer]]` backlink
- `docs/memory/MEMORY.md`, `docs/memory/skills.md` → add to breadcrumbs resources table
